1. INTRODUCTION
================================================================================

1.1 PURPOSE

The purpose of this Software Requirements Specification (SRS) document is to 
define the functional and non-functional requirements for the Blood Donation 
Management System (BDMS). The BDMS is designed to manage donation appointments, 
monitor current blood inventory stocks in real-time, keep track of all current 
registered users in the system, identify eligible donors based on their donation 
history and health criteria, and facilitate staff contact with eligible donors 
for blood donation requests.

This document provides a comprehensive description of all system features, capabilities, constraints, and interface requirements necessary to develop, test, and deploy the system.

This SRS serves as:
- A communication medium between stakeholders and the development team
- A contract for system implementation and testing
- A basis for project planning and resource allocation
- A reference document for system maintenance and future enhancements

1.4 PRODUCT SCOPE

The Blood Donation Management System is a web-based application designed to manage donation appointments, monitor current blood stock levels in real-time, keep track of current registered users, identify eligible donors based on health criteria and donation history, and facilitate contact with donors for blood donation requests. The system streamlines communication and coordination between donors, medical staff, and blood bank administrators.

A) INCLUDED IN SCOPE
   - Current user registration, tracking, and authentication for all system users
   - Donation appointment booking, scheduling, and management
   - Real-time blood inventory monitoring and stock tracking by blood type
   - Eligible donor identification based on 56-day deferral period
   - Eligibility calculation using donation history and health criteria
   - Donor contact and notification capabilities for eligible donors
   - Staff capability to search and identify eligible donors for contact
   - Donor search and filtering
   - Staff appointment calendar and timeline management
   - Session management and role-based access control

B) EXCLUDED FROM SCOPE
   - Laboratory blood testing and analysis equipment
   - Physical blood collection equipment interfacing
   - Health metrics recording and monitoring
   - Hemoglobin and blood pressure tracking
   - Donor health status management
   - Integration with external hospital management systems (Phase 2)
   - SMS notifications (email only for MVP)
   - Multi-language support (English only for v1.0)
   - Advanced reporting and analytics (basic statistics only)
   - Third-party payment processing integration
   - Biometric authentication systems
   - Backend database implementation (Phase 2)

2. OVERALL DESCRIPTION
================================================================================

2.1 PRODUCT PERSPECTIVE

The Blood Donation Management System is a standalone web-based application that 
operates independently but may be integrated with hospital management systems 
in future phases. The system serves five primary purposes: (1) manage donation 
appointments, (2) monitor current blood inventory stocks in real-time, (3) keep 
track of all current registered users in the system, (4) identify eligible 
donors based on their donation history and health criteria, and (5) facilitate 
staff contact with eligible donors for blood donation requests. The system acts 
as the central hub for coordination between donors and medical staff for all 
appointment scheduling and donor engagement activities.

2.1.1 SYSTEM ARCHITECTURE

The system is implemented as a single-page web application (SPA) with the 
following components:

A) DONOR PORTAL
   - Web-based interface for registered blood donors
   - Accessible 24/7 for appointment booking and history review
   - Self-service features for profile and appointment management
   - Real-time eligibility status tracking
   - Donation history timeline visualization

B) STAFF PORTAL
   - Web-based interface for medical personnel and administrators
   - Access restricted to authorized donation center staff
   - Features for appointment management, donor search, inventory control
   - Administrative dashboard with system statistics
   - Role-based access control for different staff levels

C) SHARED COMPONENTS
   - Authentication and session management layer
   - Email notification service integration
   - Mock data layer (current MVP)
   - RESTful API interface (for future backend)

2.1.2 DEVELOPMENT ENVIRONMENT

The system is currently implemented as an MVP using:
- React 19.2.0 for user interface
- Vite 7.3.1 as build tool and development server
- React Router v6 for client-side navigation
- Mock data stored in JavaScript objects
- localStorage for session persistence
- CSS Grid and Flexbox for responsive design

2.1.3 PRODUCTION ENVIRONMENT (PLANNED)

Future production deployment will include:
- Node.js/Express backend server
- MongoDB or PostgreSQL database
- JWT-based authentication
- Email service provider integration (SendGrid/AWS SES)
- SSL/TLS encryption for all communications
- Cloud hosting platform (AWS/Azure/GCP)
- Automated backup and disaster recovery

2.2 PRODUCT FUNCTIONS

The Blood Donation Management System provides the following primary functions:

A) CURRENT USER TRACKING AND MANAGEMENT
   - User registration and profile creation
   - Tracking of all current registered donors in the system
   - Tracking of current staff and administrative users
   - Profile maintenance and contact information management
   - User authentication and session management
   - Role-based access control for donors, staff, and administrators

B) APPOINTMENT MANAGEMENT AND SCHEDULING
   - Online appointment booking by donors
   - Multi-step booking process (date/time selection → confirmation)
   - Real-time slot availability checking and prevention of overbooking
   - Appointment scheduling and tracking by staff
   - Appointment status workflow management (pending → confirmed → checked-in → completed)
   - Weekly calendar view for staff planning
   - Daily timeline view of appointments for easy processing
   - Appointment confirmation and reminder notifications

C) CURRENT BLOOD INVENTORY MONITORING
   - Real-time tracking of current blood stock levels by type
   - Stock level alerts for low and critical inventory status
   - Automatic inventory updates based on donations processed
   - Expiration date tracking for stored blood units
   - Visual status indicators (adequate/low/critical) for quick assessment
   - Inventory overview information for staff decision-making

D) ELIGIBLE DONOR IDENTIFICATION AND TRACKING
   - Calculation and tracking of donor eligibility status
   - 56-day deferral period calculation for whole blood donations
   - Current eligible donor identification and listing
   - Eligibility countdown tracking for ineligible donors
   - Donor search and filtering by eligibility status
   - Blood type determination and storage

E) DONOR CONTACT AND ENGAGEMENT CAPABILITIES
   - Staff capability to identify and contact eligible donors
   - Donor search by multiple criteria (name, ID, email, phone)
   - Quick access to donor contact information
   - Automated appointment confirmation emails to donors
   - Automated appointment reminder emails (24 hours prior)
   - Notification system for donors regarding eligibility status
   - Staff portal for managing donor outreach communications

   - Session validation on each request (future)
   - Logout clears all session data

D) MULTI-DEVICE PREVENTION
   - Prevent concurrent logins from multiple devices (future)
   - Validate session from single IP address (future)
   - Force logout on detecting concurrent access (future)

5.3.2 Data Protection

A) DATA IN TRANSIT
   - HTTPS protocol mandatory for all internet communication
   - SSL/TLS certificate required (minimum TLS 1.2)
   - Certificate from trusted Certificate Authority
   - Certificate pinning (future for high security)
   - No sensitive data sent over plain HTTP
   - Secure WebSocket for real-time features (future)

B) DATA AT REST
   - Database passwords encrypted and stored securely
   - Sensitive fields encrypted in database (Phase 2):
     * Social security numbers
     * Blood type
   - Encryption algorithm: AES-256 or equivalent
   - Encryption keys managed securely:
     * Stored separately from encrypted data
     * Rotated regularly
     * Access controlled

C) DATA PRIVACY AND CONFIDENTIALITY
   - Donor data visible only to donor owner and authorized staff
   - No sensitive data shared with billing or administrative staff
   - No data shared with external parties without consent
   - Compliance with HIPAA Privacy Rule
   - Privacy Policy prominently displayed
   - User consent for data collection

5.5 BUSINESS RULES

BR1: DONATION ELIGIBILITY
     Blood donors must wait 56 days between whole blood donations. No donor can
     schedule or complete a donation if less than 56 days have passed since their
     last whole blood donation.

BR2: BLOOD TYPE DETERMINATION
     Blood type is not collected during donor registration. Blood type must be
     determined by qualified medical staff during the first blood donation
     appointment based on laboratory testing.

BR3: APPOINTMENT AVAILABILITY
     Appointment slots can only be booked during donation center operating hours.
     Each time slot is limited to one donor appointment maximum unless the staff
     specifically increases capacity.

BR4: INVENTORY MANAGEMENT
     Blood inventory levels must be monitored continuously. When inventory for
     any blood type drops below 10 units (critical level), the system shall
     alert staff that new donations or supplies are urgently needed.

BR5: DONATION CENTER OPERATIONS
     The donation center operates during scheduled hours and accepts appointment
     bookings during those hours. Staff must update center operating hours if
     the center will be temporarily closed.

BR6: SESSION SECURITY
     User sessions automatically expire after 30 minutes of inactivity for
     security. Users must re-authenticate to continue using the system.

BR7: DATA RETENTION
     Donor records and donation history are retained for a minimum of 5 years
     for medical and legal compliance. Records must be securely archived after
     this period.

BR8: APPOINTMENT STATUS WORKFLOW
     Appointments follow a strict status workflow: Pending → Confirmed → 
     Checked-In → Completed, or may be Cancelled at any point. Appointments
     cannot skip steps in this workflow.

BR9: PRIVACY AND CONFIDENTIALITY
      Donor personal information and donation records are private and visible
      only to the donor themselves and authorized medical staff. No other
      users or systems can access this information without explicit consent.

B.2 DATA FLOW DIAGRAM

DONOR REGISTRATION FLOW:
1. Donor enters registration form
2. System validates input
3. System creates new donor account
4. System stores in data storage
5. System sends confirmation email (Phase 2)
6. System redirects to login

APPOINTMENT BOOKING FLOW:
1. Donor verifies eligibility
2. Donor selects donation center
3. Donor selects date and time
4. System checks availability
5. Donor confirms booking
6. System creates appointment record
7. System sends confirmation email (Phase 2)
8. System redirects to dashboard

DONATION PROCESSING FLOW:
1. Staff views appointment calendar
2. Staff marks donor as checked-in
3. Staff verifies donor blood type information
4. Staff marks donation as completed
5. System creates donation record
6. System calculates next eligible date
7. System decrements inventory
8. System sends completion notification (Phase 2)
